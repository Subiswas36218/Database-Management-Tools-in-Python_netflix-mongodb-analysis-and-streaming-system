# Report — Netflix Database Analysis and Messaging System

This report summarizes the Netflix dataset analysis and messaging system implementation using MongoDB Atlas and Python.

## Contents
- Dataset preprocessing
- Database comparison
- MongoDB selection
- Aggregation queries
- Visualizations
- Queue-based messaging system
- Reflection and conclusion

## Selected Database
MongoDB Atlas was selected because:
- flexible schema
- arrays support
- aggregation pipelines
- scalability

## Queries Performed
1. Most common genres
2. Top content-producing countries
3. Netflix titles released over time

## Messaging System
Implemented:
- SimpleTopic
- SimpleProducer
- SimpleConsumer
- AdvancedTopic
- AdvancedProducer
- AdvancedConsumer

Queue/FIFO behaviour was used.
